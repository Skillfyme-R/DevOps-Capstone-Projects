# API Reference

Base path: `/api/pulsar/v1`. All examples assume `pulsar-server` running at `http://localhost:8080`.

A live, interactive OpenAPI/Swagger UI is generated by springdoc and served at
`/swagger-ui.html` (raw spec at `/v3/api-docs`) whenever the server is running — use it to
try requests directly against a live instance. This document is the static reference, read
directly from the controllers and DTOs in `pulsar-server/src/main/java/.../api/v1/`.

## Authentication

Two schemes, enforced by `SecurityConfig` (path-based, see [`security.md`](security.md) for the
full model):

- **Bearer JWT** — `Authorization: Bearer <accessToken>`, obtained from `/auth/login`. Used by
  human operators (`ADMIN`, `OPERATOR`, `VIEWER` roles).
- **API key** — `X-Pulsar-Api-Key: <key>`, grants the `WORKER` role. Used by worker SDK clients.

| Role | Can do |
|---|---|
| `ADMIN` | Everything |
| `OPERATOR` | Register definitions, start/pause/resume/terminate/retry/rerun workflows, register task definitions |
| `WORKER` | Poll/complete/fail/extend-lease on `/tasks/**` only |
| `VIEWER` | Any `GET` under `/api/pulsar/v1/**` |

## Auth

### `POST /api/pulsar/v1/auth/login`

**Auth**: none. **Body** (`LoginRequest`):
```json
{ "username": "admin", "password": "PulsarAdmin123!" }
```
**Response** `200 OK` (`TokenResponse`):
```json
{ "accessToken": "...", "refreshToken": "...", "tokenType": "Bearer" }
```
`401` with `errorCode: PULSAR-2001` (`AUTHENTICATION_FAILED`) on bad credentials.

### `POST /api/pulsar/v1/auth/refresh`

**Auth**: none. **Body** (`RefreshTokenRequest`): `{ "refreshToken": "..." }`.
**Response** `200 OK` (`TokenResponse`) — a fresh access + refresh token pair. `401` /
`PULSAR-2004` (`TOKEN_EXPIRED`) if the refresh token itself has expired.

## Workflow Definitions

### `POST /api/pulsar/v1/workflow-definitions`

**Auth**: `OPERATOR` or `ADMIN`. **Body** (`RegisterWorkflowDefinitionRequest`):
```json
{
  "name": "video-ingest-pipeline",
  "version": 1,
  "description": "...",
  "tasks": [ /* array of TaskDefinitionRefDto, see below */ ],
  "timeoutSeconds": 7200,
  "ownerEmail": "media-ops@reelforge.media"
}
```
`TaskDefinitionRefDto` shape (mirrors `pulsar-core`'s `TaskDefinitionRef`):
```json
{
  "taskReferenceName": "validate_upload_ref",
  "taskType": "SIMPLE",
  "name": "validate-upload",
  "inputParameters": { "assetId": "${workflow.input.assetId}" },
  "next": ["fork_transcode_thumbnail_ref"],
  "joinOn": [],
  "decisionInputParameter": null,
  "decisionCases": {},
  "defaultCase": [],
  "subWorkflowName": null,
  "subWorkflowVersion": 0
}
```
**Response** `201 Created` (`WorkflowDefinitionResponse`) — echoes the saved definition including
generated `id` and `createdAt`. `400` / `PULSAR-1005` (`INVALID_WORKFLOW_DEFINITION`) or
`PULSAR-1010` (`CYCLIC_WORKFLOW_GRAPH`) if `WorkflowGraphValidator` rejects the DAG (dangling
reference, unmatched FORK/JOIN, or a cycle). `409` / `PULSAR-1006`
(`DUPLICATE_WORKFLOW_DEFINITION`) if `(name, version)` already exists.

### `GET /api/pulsar/v1/workflow-definitions/{name}`

**Auth**: any authenticated role (including `WORKER`). Returns the **latest** version by name.
`404` / `PULSAR-1001` if no definition with that name exists.

### `GET /api/pulsar/v1/workflow-definitions/{name}/versions/{version}`

**Auth**: any authenticated role. Returns a specific version.

### `GET /api/pulsar/v1/workflow-definitions/{name}/versions`

**Auth**: any authenticated role. Returns all registered versions of a name as a list.

> There is no endpoint to list all *distinct workflow definition names* — you must already know
> the name to look it up. See [Known Limitations](../README.md#known-limitations).

## Workflow Executions

### `POST /api/pulsar/v1/workflows`

**Auth**: `OPERATOR` or `ADMIN`. **Body** (`StartWorkflowRequest`):
```json
{ "workflowName": "video-ingest-pipeline", "version": null, "input": { "assetId": "..." }, "correlationId": "optional" }
```
`version: null` resolves to the latest registered version. **Response** `201 Created`
(`StartWorkflowResponse`): `{ "workflowExecutionId": "<uuid>" }`. `404` / `PULSAR-1001` if the
named (and optionally versioned) definition doesn't exist.

### `GET /api/pulsar/v1/workflows/{id}`

**Auth**: any authenticated role. Returns `WorkflowExecutionResponse`:
```json
{
  "id": "uuid", "workflowDefinitionId": "video-ingest-pipeline", "workflowDefinitionVersion": 1,
  "status": "RUNNING", "input": {}, "output": {}, "startTime": "...", "endTime": null,
  "correlationId": null,
  "tasks": [ /* TaskExecutionResponse[] — every task execution for this run */ ]
}
```
`status` is one of `RUNNING`, `PAUSED`, `COMPLETED`, `FAILED`, `TERMINATED`, `TIMED_OUT`. `404` /
`PULSAR-1003` if the id doesn't exist.

### `PUT /api/pulsar/v1/workflows/{id}/pause`

**Auth**: `OPERATOR`/`ADMIN`. `204 No Content`. `409` / `PULSAR-1007`
(`ILLEGAL_STATE_TRANSITION`) if the workflow isn't `RUNNING`.

### `PUT /api/pulsar/v1/workflows/{id}/resume`

**Auth**: `OPERATOR`/`ADMIN`. `204 No Content`, then immediately calls `progressWorkflow()` so any
tasks that would have advanced while paused get scheduled right away. `409` if not `PAUSED`.

### `PUT /api/pulsar/v1/workflows/{id}/terminate`

**Auth**: `OPERATOR`/`ADMIN`. `204 No Content`. Valid from `RUNNING` or `PAUSED`; `409` from any
terminal status.

### `POST /api/pulsar/v1/workflows/{id}/retry`

**Auth**: `OPERATOR`/`ADMIN`. Reschedules every task currently `FAILED` or `TIMED_OUT` back to
`SCHEDULED` (incrementing `retryCount`) and re-pushes them onto the queue; if the workflow itself
was `FAILED`, flips it back to `RUNNING`. `204 No Content`.

### `POST /api/pulsar/v1/workflows/{id}/rerun`

**Auth**: `OPERATOR`/`ADMIN`. Starts a **brand-new** `WorkflowExecution` against the same
definition version and original input/correlation id (does not touch the original execution).
`201 Created` (`StartWorkflowResponse`) with the new execution's id.

## Task Definitions

### `POST /api/pulsar/v1/task-definitions`

**Auth**: `OPERATOR`/`ADMIN`. **Body** (`RegisterTaskDefinitionRequest`):
```json
{
  "name": "transcode-video", "description": "...", "retryCount": 3,
  "retryLogic": "EXPONENTIAL_BACKOFF", "retryDelaySeconds": 10, "timeoutSeconds": 1800,
  "inputKeys": ["assetId"], "outputKeys": ["renditions"]
}
```
`retryLogic` is one of `FIXED`, `LINEAR_BACKOFF`, `EXPONENTIAL_BACKOFF`. **Response**
`201 Created` (`TaskDefinitionResponse`).

### `GET /api/pulsar/v1/task-definitions/{name}`

**Auth**: any authenticated role. `404` / `PULSAR-1002` if not found.

## Tasks (worker-facing)

This is the surface a `TaskWorker`/`PulsarApiClient` actually calls — analogous to Conductor's
worker task API.

### `GET /api/pulsar/v1/tasks/poll/{taskType}?workerId={workerId}`

**Auth**: `WORKER`/`OPERATOR`/`ADMIN`. `taskType` here is the **queue topic**, which is the DAG
node's `taskReferenceName`, not the reusable `TaskDefinition.name` — a `TaskWorker`'s
`getTaskType()` must return the workflow's `taskReferenceName` string (the bundled demo workers all
do this correctly). **Response**: `200 OK` with a
`TaskExecutionResponse` if work was available (status flips to `IN_PROGRESS`, `workerId` and
`startTime` recorded, a 60-second lease — `PULSAR_DEFAULT_LEASE_SECONDS` — begins); **`204 No
Content`** if the queue was empty.

### `POST /api/pulsar/v1/tasks/{id}/complete`

**Auth**: `WORKER`/`OPERATOR`/`ADMIN`. **Body** (`CompleteTaskRequest`): `{ "output": { ... } }`.
Marks the task `COMPLETED`, acknowledges the queue lease, and triggers
`WorkflowExecutionService.progressWorkflow()` to schedule whatever comes next. `204 No Content`.
`409` / `PULSAR-1007` if the task isn't `IN_PROGRESS`.

### `POST /api/pulsar/v1/tasks/{id}/fail`

**Auth**: `WORKER`/`OPERATOR`/`ADMIN`. **Body** (`FailTaskRequest`):
```json
{ "reason": "transcode farm returned non-zero exit", "terminal": false }
```
If `terminal: true`, or the task's registered `TaskDefinition.retryCount` is exhausted, the task
goes straight to `FAILED_WITH_TERMINAL_ERROR` (which fails the whole workflow on the next
`decide()`). Otherwise it's marked `FAILED` then immediately re-`SCHEDULED` with `retryCount + 1`
and a delay from the task's `RetryPolicy` (`FixedRetryPolicy`, `LinearRetryPolicy`, or
`ExponentialBackoffRetryPolicy`, capped at 3600s) baked into `scheduledTime` — then re-pushed onto
the queue right away (the queue has no delayed-delivery primitive; see
[Known Limitations](../README.md#known-limitations)). `204 No Content`.

### `PUT /api/pulsar/v1/tasks/{id}/lease`

**Auth**: `WORKER`/`OPERATOR`/`ADMIN`. **Body** (`ExtendLeaseRequest`): `{ "extensionSeconds": 30 }`.
Extends both the Redis ZSET lease score and the persisted `callbackAfterSeconds` (which is what
`TaskTimeoutSweeper` compares against). Only valid while the task is `IN_PROGRESS`; otherwise
`409` / `PULSAR-1007`.

## Webhooks

### `POST /api/pulsar/v1/webhooks/{source}`

**Auth**: none (public — this is the point). Looks up `source` in the in-memory
`EventHandlerRegistry` to resolve a workflow name, then starts that workflow with the request body
(or `{}` if none) as input and `correlationId = "webhook:{source}"`. `201 Created`
(`StartWorkflowResponse`). `404` / `PULSAR-1001` (mapped from `WORKFLOW_DEFINITION_NOT_FOUND`, the
error code `EventHandlerRegistry` reuses for "no handler registered") if `source` isn't mapped.
There is no CRUD API to register a `source → workflowName` mapping — it must be added
programmatically (`EventHandlerRegistry.register(...)`), and mappings are lost on restart.

## Error response shape

Every non-2xx response (via `GlobalExceptionHandler`) returns:
```json
{
  "errorCode": "PULSAR-1003",
  "message": "No workflow execution found for id ...",
  "timestamp": "2026-01-01T00:00:00Z",
  "path": "/api/pulsar/v1/workflows/...",
  "fieldErrors": []
}
```
`fieldErrors` is populated (with `{ field, message }` entries) only for `400` validation failures
from `@Valid` request bodies (`errorCode: PULSAR-3001`).

### Full error code reference

| Code | Meaning | HTTP status |
|---|---|---|
| `PULSAR-1001` | Workflow definition not found | 404 |
| `PULSAR-1002` | Task definition not found | 404 |
| `PULSAR-1003` | Workflow execution not found | 404 |
| `PULSAR-1004` | Task execution not found | 404 |
| `PULSAR-1005` | Invalid workflow definition (dangling ref / unmatched FORK-JOIN) | 400 |
| `PULSAR-1006` | Duplicate workflow definition (name+version exists) | 409 |
| `PULSAR-1007` | Illegal state transition | 409 |
| `PULSAR-1008` | Task queue empty | 204 |
| `PULSAR-1009` | Task lease expired | 409 |
| `PULSAR-1010` | Cyclic workflow graph | 400 |
| `PULSAR-2001` | Authentication failed | 401 |
| `PULSAR-2002` | Authorization denied | 403 |
| `PULSAR-2003` | Invalid API key | 401 |
| `PULSAR-2004` | Token expired | 401 |
| `PULSAR-3001` | Request validation failed | 400 |
| `PULSAR-3002` | Rate limit exceeded | 429 |
| `PULSAR-5000` | Internal error (catch-all) | 500 |

(Full list from `pulsar-common`'s `ErrorCodes`; not every code is currently thrown from a
reachable code path — e.g. `PULSAR-3002` is reserved but no rate limiter is wired up yet.)
